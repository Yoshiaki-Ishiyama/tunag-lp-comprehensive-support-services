# content.md — industry(業界別 TUNAG ハブページ / 大辞典型)

業界別 LP の入り口となるハブページ。**大辞典型の索引ページ**として位置づける。Hero は CTA を置かず辞典の表紙のようにシンプルに、その下に業界グリッド・仕様の柔軟性訴求・事例/資料リンクの順に並べる。インラインCTA帯は持たない(訪問者の動線は業界別 LP への遷移、または事例/資料への遷移に集約)。

## 4. Specs(p-specs — どんな業界でも全従業員で使える仕様)

**配置意図**: 業界グリッドで「業界別 LP」へ進まない訪問者向けの、TUNAG の汎用仕様訴求。**3項目を横並び**でコンパクトに見せ、セクションの縦幅は短めに。

- **eyebrow**: `Universal Design`
- **title**: どんな業界でも、全従業員が使える設計です。
- **lead**: 業界・職種・働き方に左右されず、現場の最後の1人まで届く基盤を仕様で担保しています。
- **layout**: モバイル 1 列 / タブレット以上 3 列。カードなし、アイコン円 + 見出し + 短文の Features 風コンパクト構成。
- **items**(3 項目):
  - **icon**: device(スマホ + PC を表すアイコン)
    **title**: スマホでもPCでも使える
    **text**: 現場社員はスマートフォン、本部はPC。それぞれの業務環境に合った操作性で、同じ情報基盤を共有できます。
  - **icon**: shield(セキュリティを表すアイコン)
    **title**: 社用アドレスがない従業員にもセキュアにアカウントを発行できる
    **text**: 個人メール不要の招待方式に対応。情報セキュリティを保ちながら、現場社員一人ひとりまでアカウントを行き渡らせます。
  - **icon**: settings(権限設計を表すアイコン)
    **title**: 閲覧や制度利用の権限設計を柔軟にできる
    **text**: 役職・部署・拠点ごとに、見える情報・使える機能を細かく制御。多層的な組織にも、ガバナンスを保ったまま展開できます。

## ブランドグラデ全面背景の使用回数

**Hero のみ = 1 回**(design.md §2 配色ルール上限の 4 回以内、シンプルに倒した)。インラインCTA帯は廃止。

## 制作上のメモ

- 「導入実績」表現は使わない。「利用中企業1,400社以上」で統一
- インラインCTA帯(p-cta)・最終CTA(p-final-cta)・Contact フォームは持たない
- お問い合わせ動線は **ヘッダー右の「無料デモを見る」ボタン**と **フッターの「お問い合わせ」リンク**(リンク先は外部 / 別ページ想定で `#` プレースホルダ)
- Specs セクションは縦幅を抑えて、業界グリッドと Resources の間の「軽い橋渡し」として機能させる
- Resources セクションはボタン2つだけのシンプル構成。リッチなカードや数字訴求は事例ページ側で表現
